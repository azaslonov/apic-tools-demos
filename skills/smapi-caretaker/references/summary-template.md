# Summary Template

Format for daily caretaker summary.

## CRITICAL: File and Teams Post Must Be Identical

The Teams post content **MUST be exactly the same** as the markdown file content:
- Include ALL per-issue sections with stack traces
- Include ALL investigation queries  
- Include the full Work Items table with complete titles
- Include the "Issues Not Requiring Bugs" section
- Do NOT abbreviate or summarize for Teams

Teams supports full markdown rendering. Post the complete document.

---

## Output Structure

Use `#` headers with emoji markers for clear visual separation in Teams:

    # SMAPI/Control Plane Daily Summary - {YYYY-MM-DD}
    
    **Time Range:** {startTime} to {endTime} UTC
    
    # 📊 Executive Summary
    
    | Component | Status | Worst SLA | Regions Affected |
    |-----------|--------|-----------|------------------|
    | SMAPI SKUv1 | ✅ / ⚠️ | {value}% | {count} regions |
    | SMAPI SKUv2 | ✅ / ⚠️ | {value}% | {count} regions |
    | SMAPI Consumption | ✅ / ⚠️ | {value}% | {count} regions |
    | RP SLA | ✅ / ⚠️ | {value}% | {count} regions |
    | RP Only SLA | ✅ / ⚠️ | {value}% | {count} regions |
    
    {2-3 sentence overview: overall health, bugs created, key findings}
    
    ---

**If all healthy (SLA ≥ 99.9% everywhere):**

    # ✅ All Systems Healthy
    
    No SLA issues detected. All components maintained ≥ 99.9% SLA across all regions.
    
    ---
    
    _Generated by SMAPI Caretaker Agent on {timestamp}_

## Status Indicators

| Icon | Meaning |
|------|---------|
| ✅ | Healthy (SLA ≥ 99.9%) or bug created/updated |
| ⚠️ | SLA issue investigated |
| ❌ | No action taken (with reason) |

---

## Action Taken Format

For each issue, use one of:
- ✅ **Bug #{id} created** ([View](https://dev.azure.com/msazure/One/_workitems/edit/{id})) - Assigned to GitHub Copilot
- ✅ **Existing Bug #{id}** ([View](https://dev.azure.com/msazure/One/_workitems/edit/{id})) - Already tracked
- ⏭️ **Skipped** - {Reason: e.g., "3 bug limit reached", "Customer errors only"}
