## 🔴 Issue 1: NullReferenceException in TenantValidator

**Test**: `ResourceProvider.IntegrationTests.SKUv2.TenantTests.ValidateTenant`
**Category**: SKUv2
**Classification**: Logic Bug

### Error Details

**Error Message**:
```
System.NullReferenceException: Object reference not set to an instance of an object.
```

**Stack Trace**:
```
at Microsoft.Azure.ApiManagement.ResourceProvider.Validation.TenantValidator.Validate(TenantContext context)
at Microsoft.Azure.ApiManagement.ResourceProvider.Controllers.ApiServiceController.CreateAsync()
```

### Root Cause Analysis

The TenantValidator.Validate() method assumes TenantContext.TenantId is always populated, but for legacy subscriptions, TenantId can be null. Line 145 calls `tenantId.ToString()` without null check.

**Evidence**:
- Stack trace shows NullReferenceException at TenantValidator.Validate()
- Recent commit abc123 modified TenantValidator.cs
- Test uses a legacy subscription without TenantId

### Kusto Query to Investigate

```kql
Orchestration
| where PreciseTimeStamp > ago(2h)
| where operationName contains "CreateApiService"
| where exception contains "NullReferenceException"
| project PreciseTimeStamp, serviceName, exception
```

### Action Taken

✅ **Bug #36789012 created** ([View](https://dev.azure.com/msazure/One/_workitems/edit/36789012)) - Assigned to GitHub Copilot
